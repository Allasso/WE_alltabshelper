Ready or not, here it comes.
All Tabs Helper has now switched to WebExtensions. Perhaps unleashed in a not-fully-ready state and admittedly a little bit buggy, it is being released now to (hopefully) be in time for the release of Firefox 57, when the legacy version will stop working altogether.
Usage
All Tabs Helper can be accessed two ways, as before. By using a drop-down menu, or viewing in the sidebar. These are implemented a little differently however.
Drop-down: Rather than clicking on the "All tabs" button as before, you will now need to click on the All Tabs Helper toolbar button (The "A" icon) for the menu to appear.
Sidebar: Select All Tabs Helper in View > Sidebar.
What To Expect
The new phase of All Tabs Helper will have some noticeable changes in features and implementation. Some are intentional [1], Some are a result of differences in the way WebExtensions for implements UI, some are held up by the current unavailability of an API to support them [2], and some will probably never be implemented due to the fact that mozilla does not intend on providing APIs that would support the features desired.
[1] With this round of ATH, I plan to keep things simple. Hence I will not be implementing many of the custom conveniences I have in the past. I may not even implement some features even if APIs eventually are provided to support them. You may notice the options page strikingly stark. I will not bother to list what conveniences or features will be missing.
My main focus with ATH has always been to:
Allow users to operate on tabs from a vertical menu.
Allow closing and moving of multiple tabs.
Filter tabs by title/url.
Quickly find duplicate tabs.
List tabs in order of the most recently accessed.
Anything more than that is a bonus.
[2] There are plans to **possibly** implement the following familiar features if APIs become available:
Open new tab next to current: Mozilla bug 1344749 (https://bugzilla.mozilla.org/show_bug.cgi?id=1344749). Note that it is possible this feature will be introduced as a Firefox browser preference, which will negate the need of having this as an ATH feature.
Unload tab(s): Mozilla bug 1322485 (https://bugzilla.mozilla.org/show_bug.cgi?id=1322485). This bug has landed in FF 58, and should be available in that release. Note that this feature only works on multi-process tabs.
Search tabs content: This feature is currently available though in a fledgling state. Expect some bugs, particularly if searching complex pages with frames. PDFs are presently not searchable using this feature. Maybe improvements will come in the future, maybe not. This feature is after all, outside the scope mentioned in [1].
My View on What Mozilla is Doing
I would like to say that I fully support Mozilla's decision to switch to WebExtensions. This has been a hot and heavy debate among users and addon developers, and I refrain from going into detail here. But I honestly believe that it is a carefully thought-out decision, and not simply Mozilla trying to imitate Chrome, as many assert. All in all I see it as a move forward that in the longrun will be a great improvement. It may not be obvious to users (or even addon devs) the benefits of secure browsing and future development of the Firefox web browser gained by creating an API driven environment for addons. Sometimes an unruly tree must be pruned in order to eventually result in a more flourishing and fruitful one, and that pruning can be painful for a time.
